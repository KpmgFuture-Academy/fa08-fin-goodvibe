# 팀 기여 정리

이 문서는 최종 발표 스크립트 v3와 `locaville` 구현 내용을 기준으로 포트폴리오용 역할을 정리한 것입니다.

## 박찬호

- 팀장 및 최종 발표
- 문제정의, 현장 리서치, ESG/AX 관점 정리
- 농업인 앱 UX 흐름 설계
- `app_user` 중심 프론트엔드 구현
- 오늘 할 일, 영농일지 단일 진입점, 라이브 사진 코칭, 도우미 모드 흐름 구현/연동
- AI 기능 연동: Vision 코칭, STT/TTS, 오늘 한마디, RAG 도움말 사용 흐름
- 백엔드 API 연동 및 데모 시나리오 구성

## 강창희

- FastAPI 백엔드 구조 설계 및 구현
- PostgreSQL 중심 DB 모델링과 repository 계층 구현
- `identity_rdb` 기반 farmer_id 정규화 흐름
- RAG 파이프라인, 정책 문서 ingest, 정책 Q&A 구조
- To-do, 영농일지, 증빙자료, 사업/활동 API 데이터 흐름 정리
- DB 권위 원칙, AI advisory 원칙, computed_status 설계

## 박순선

- QA 및 테스트 산출물 정리
- 요구사항, 테스트 시나리오, 테스트 결과 문서 관리
- 고령 사용자 기준 UX 점검
- 발표/시연 산출물 정리 보조
- 기능별 검수 관점 정리: 영농일지, 증빙, 알림, 이장님 검토, 관리자 화면

## 이동현

- 초기 기획 및 문제정의 보조
- 아이디어 발산과 프로젝트 방향성 정리 지원

## 협업 방식

- 5개 sprint 흐름으로 POC -> MVP -> 배포/시연 -> 발표 산출물을 구성
- GitHub, 문서, 데모 seed, 요구사항정의서, 테스트 결과를 함께 관리
- 핵심 의사결정은 "어르신이 실제로 쓸 수 있나"와 "DB가 진실, AI는 보조"를 기준으로 정리
