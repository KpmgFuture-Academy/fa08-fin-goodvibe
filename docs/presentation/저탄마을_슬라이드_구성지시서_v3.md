# 저탄마을 — Claude Design 슬라이드 지시서 v3 (본편 13장)

> 팀 합의 재구성 반영. 부록 A1–B10은 **현행 렌더 유지**(이 지시서는 본편만). 본편을 이 13장 기준으로 다시 빌드.

## 글로벌 디자인

- **포맷:** 16:9 · Pretendard
- **팔레트:** bg `#FAF7F0` · ink `#1A2B20` · green-deep `#14402A` · green-mid `#1F6B3F` · green-tint `#E8F0E6` · gold `#E8A93D`(강조 1개만) · muted `#5A6B5E` · line `#D9D4C8`
- **헤더 시스템:** 좌상단 "저탄마을" + 페이지번호, 그 아래 작은 카테고리, 큰 슬라이드 타이틀. (부록과 동일 패턴)

## 탈-AI 원칙 (이번 리빌드의 핵심)

1. **한 슬라이드 = 한 메시지.** 카드 3개로 쪼개는 습관 줄이기.
2. **3칸 평행 대칭 깨기.** 모든 게 1/3·1/3·1/3이면 기계 냄새. 강약(1+강조+1, 배너+요소) 주기.
3. **여백과 위계로 말하기.** 박스·테두리 줄이고 타이포 크기 차로 정리.
4. **골드(`#E8A93D`)는 슬라이드당 1곳만** — 진짜 강조할 한 줄에.
5. 화살표(→)·가운뎃점(·) 남발 자제. 꼭 필요한 흐름에만.

## S1. 훅 — 매일 기록, 쓰시나요?
- **목적:** 청중 공감 → 농업인 전환. 클로징과 북엔드.
- **레이아웃:** 거의 빈 화면. 중앙 큰 질문 한 줄. 하단 작게 발표자.
- **카피:** (대) "매일 기록, 쓰시나요?" / (소) "굿바이브 · 박찬호"
- **비주얼:** 텍스트만. 배경 bg. 장식 최소(아주 옅은 라인 한 줄 정도).
- **노트:** 카드·아이콘 금지. 여백이 메시지. "회고" 단어 쓰지 말 것.

## S2. 문제 — 영농일지는 증빙이다
- **목적:** 영농일지=돈(증빙)임을 박기.
- **레이아웃:** 상단 큰 단언 한 줄 + 그 아래 `기록 → 증빙 → 지원사업` 가로 흐름(작게). 우하단에 보도 인용 한 줄.
- **카피:** (대) "이 기록은 일기가 아닙니다 — 친환경 지원사업의 **증빙**입니다." / (중) "기록·증빙이 누락되면, 받을 지원에서 불이익." / (인용, 작게) "영농일지가 '숙제'로 여겨진다" — 농정 언론
- **비주얼:** 흐름 3박스는 아주 가볍게. 인용은 따옴표 스타일.
- **노트:** 단언 문장이 주인공. 흐름·인용은 보조 크기.

## S3. 리빌 — 사진 한 장, 목소리 한 마디
- **목적:** 제품 약속 + 실제 '오늘 할 일' 카드 미리보기.
- **레이아웃:** 좌측 카피, 우측 폰 목업 안에 '오늘 할 일' 카드(비대칭 2분할).
- **카피:** (좌·대) "영농일지와 증빙을, **사진 한 장과 목소리 한 마디**로." / (우 카드) 오늘 우선: 중간 물떼기 · 위치 1번 논 · 마감 7/11(27일 남음) · 증빙: 작업 사진 필요 · "작업한 곳이 잘 보이게 찍어주세요" · [사진 찍고 완료하기]
- **비주얼:** 폰 목업 권장(데모의 예고편). 카드는 green-tint 배경 + gold 버튼 1개.
- **노트:** 카드가 진짜 앱처럼 보이게(실물감).

## S4. 현장 리서치
- **목적:** 진정성·신뢰. 도우미 복선.
- **레이아웃:** 상단 타이틀 "전화기를 들고 현장을 확인했습니다" + 인용 2개(엇갈리게 배치, 대칭 X) + 하단 한 줄.
- **카피:** 인용① "저탄소 농업프로그램? 그런 게 있는 줄도 몰랐는데…" — 팀원 고향 이장님 / 인용② "디지털 격차가 심해서, 기록 보조 도우미가 따로 있어요." — 동네 복지센터 / (하단) "현장에서, 사람에게서 시작했습니다."
- **비주얼:** 인용 2개를 같은 크기·같은 자리로 두지 말고 살짝 엇갈리게(손으로 붙인 메모 느낌).
- **노트:** "기록 보조 도우미"가 뒤(S10) 복선 — 시각적으로 살짝 강조.

## S5. As-Is → To-Be 통합 〔재구성 핵심〕
- **목적:** 3난제→3해결을 *한 장에* (지침→기록→디지털 순). As-Is 슬라이드 흡수.
- **레이아웃:** 좌 As-Is | 우 To-Be, 3행. **가운데(기록) 행을 가장 크게**(1+강조+1 비대칭). 좌상단 작게 "As-Is", 우상단 "To-Be" 라벨.
- **카피(행별):**
  - 지침이 어렵다 → 지침에서 '할 일'을 추출해 전달 *(개인에겐 '오늘 할 일'만)*
  - **기록이 버겁다 → 사진 라이브 코칭 + 음성 인식** *(찍으면서 안내받고, 말하면 일지가 된다)* ★
  - 디지털이 낯설다 → 이장님·이웃·기록 도우미가 함께 *(정말 어려운 분은 도우미가 대신)*
- **비주얼:** 가운데 행만 카드 크게/배경 green-tint, 위아래 행은 한 줄 텍스트로 가볍게. 화살표 1개 스타일 통일.
- **노트:** 세 행이 후반에서 회수됨(지침→S8, 기록→S9, 디지털→S10) — 디자인상 표시할 필욘 없지만 순서 고정. 디지털 행이 '포용'으로 따뜻하게 끝나게.

## S6. 서비스 흐름 — 세 사람, 하나의 카드
- **목적:** 데모 전 역할 오리엔테이션.
- **레이아웃:** 가로 3스텝 흐름(운영자 → 이장님 → 농업인), 마지막(농업인 카드)에 무게.
- **카피:** 헤드라인 "복잡한 지침이, 단 하나의 '오늘 할 일'로" / 운영자: 사업·활동·작업 정의 / 이장님: 마을 농가·사업 연결 / 농업인: 기준·일정 붙은 '오늘 할 일' 카드 도착
- **비주얼:** 앞 둘은 작게, 마지막 농업인 카드를 크게(결과 강조). 균일 3박스 피하기.
- **노트:** 이장님을 여기서 소개(데모·S10에서 재등장).

## S7. 시연
- **목적:** 증명. 2분 영상.
- **레이아웃:** 상단 "직접 보시죠" + 볼 포인트 3 + 큰 영상 영역.
- **카피:** ① 사진을 어떻게 찍는지 ② 이장님이 어떻게 확인하는지 ③ 어르신이 얼마나 적게 입력하는지 / (영상 자리) 시연 · 약 2:00
- **비주얼:** 영상 영역이 주인공(크게). 볼 포인트 3개는 상단 작은 띠.
- **노트:** 영상 파일 교체. 발표자가 "실제로 돌아가는 앱"이라고 멘트.

## S8. to-do 산정 — 간략 ERD 〔재구성: 곱셈식 제거 · "확정은 사람이" 흡수〕
- **목적:** '오늘 할 일'이 데이터로 어떻게 생기는지 + 합성키 추적성 + 거버넌스 원칙 한 줄.
- **레이아웃:** 상단 3단계 흐름(읽는다→다듬는다→만든다), 가운데 산정 법칙, 하단 2줄(가치·원칙).
- **카피:**
  - STEP 1 **AI가 읽는다**(초안만) — 지침에서 할 일 초안 추출
  - STEP 2 **사람이 다듬는다** — 관리자: 작업 기준 / 이장님: 참여 농가
  - STEP 3 **시스템이 만든다** — **작업 기준 × 참여 농가 → 농가별 to do**
  - (간략 엔티티) 사업 → 활동 → 작업 기준 · 참여단체 → 참여 농가
  - (작은 배지 1개) `todo_id = 경영체·사업·활동·순번` → "ID만 봐도 역추적"
  - (하단 가치) "사람이 엑셀로 짜면 며칠 — 우리는 버튼 한 번"
  - (하단 원칙, **골드**) "AI는 초안만, 확정은 사람이."
- **비주얼:** **간략 ERD**(5~6박스, 부록 A2보다 단순). 곱셈 숫자식(10×2×4) **금지**. 합성키는 *작은 ID 배지 한 개*로만 암시(전체 스키마·필드 나열 X — 그건 부록 A2).
- **화면 vs 말로:** *화면엔* 3단계 + 산정법칙 + ID 배지 + 원칙 한 줄까지만. *말로* 풀 것 — 합성키로 누가·어떤 사업·어떤 작업까지 역추적된다는 점, "자동 저장 없음·사람이 누를 때만 INSERT"라는 DB 권위. (슬라이드에 문장으로 넣지 말 것)
- **노트:** 골드는 마지막 원칙 한 줄에만. 이 슬라이드가 옛 '핵심구조+설계철학' 두 장을 합친 것. 발표자가 추적성·DB권위를 1:10 동안 내레이션하므로 슬라이드는 *앵커*만.

## S9. 라이브 코칭 — 진짜 어떻게 도는가 〔1:30, authenticity 주력〕
- **목적:** "진짜 만들었나" 의심 차단 ① + 비용 낮음. 이 덱에서 가장 깊은 기술 슬라이드.
- **레이아웃:** 가로 파이프라인 + **조건 분기**가 핵심. 일직선 흐름이 아니라, 가운데 '판단 분기'가 보이게.
- **카피(흐름):**
  - `카메라 프레임` → `앱: 8×8 픽셀 서명 비교`
  - 분기 ① **변화 거의 없음** → *로컬 안내만, Vision 미호출, 간격 2초→4초* (점선·회색)
  - 분기 ② **화면 바뀜** → `/photo-guard/coach` → `Vision(Gemini Flash Lite)` → `TTS(Chirp 3 HD)` → "조금 더 가까이" 음성 (실선·강조)
  - 루프백 표시: **ok 3연속 → 폴링 정지** / 움직이면 재개
  - (작은 태그, Vision 박스에) "시행지침 기준(evidence_type × job_cd) 주입"
- **비주얼:** **분기가 주인공.** "안 부르는 경로(회색·점선)"와 "부르는 경로(녹색·실선)"의 대비가 한눈에. 카메라→음성까지 시각적으로 멋지게. 모델명·엔드포인트는 작게.
- **화면 vs 말로:** *화면엔* 분기 다이어그램 + 핵심 라벨(8×8 서명 / 2초→4초 / ok 3연속 정지 / 지침 기준 주입)만. *말로* 풀 것 — 왜 폴링이 비용 문제인지, 서명 비교로 어떻게 거르는지, 프롬프트 주입의 의미. 슬라이드에 설명 문장 X.
- **노트:** 하단에 비전문가용 한 줄 꼭 — **"화면이 멈춰 있으면 돈이 안 나간다 / 어르신은 카메라만 들면 된다."** 이 so-what이 기술 구간의 숨구멍. 발표자가 1:30 내레이션하므로 슬라이드는 분기 구조만 또렷하게.

## S10. 도우미 · 이장님 확인 〔1:05〕
- **목적:** 포용(디지털 약속 회수) + 거버넌스를 *행동으로*.
- **레이아웃:** 좌우 2흐름(포용 | 거버넌스), 하단 한 줄. 좌우 톤 대비.
- **카피:**
  - (좌·포용) 정말 어려운 분 → 이장님·이웃·기록 도우미가 대신 사진·음성 / *작은 태그:* "시행지침의 '이행증빙도우미'와 정합"
  - (우·거버넌스) 모든 증빙 → 이장님 재확인 → 확정 *(AI 단독 결정 없음)* / *작은 태그:* "위험통과(false_OK) 0% + 사람 확정"
  - (하단) "한 명도 빠지지 않게 · **확정은 사람이**"
- **비주얼:** 좌(사람들이 둘러싼 따뜻한 톤) vs 우(검토 체크 흐름, 절제된 톤). 대비 살리기.
- **화면 vs 말로:** *화면엔* 두 흐름 + 두 작은 태그(이행증빙도우미 / false_OK 0%)만. *말로* 풀 것 — 정책에 도우미 역할이 명시돼 있어 시스템으로 받았다는 점, "정확도가 아니라 위험통과율을 0%로 맞췄다"는 판단 기준.
- **노트:** false_OK 0%(S11)의 의미가 여기서 살아남. 추상 격언 X, 작동 장면 O. '책임 있는 AI'를 *말로* 명시.

## S11. 모델 선택 + 벤치마크 — 데이터로 골랐다 〔1:15〕
- **목적:** AI를 데이터로 골랐다(기술 임팩트=숫자) + "진짜 만들었다" 못박기.
- **레이아웃:** 상단 가로 배너(토대=데이터 모델·합성키) + 그 위에 AI 3지표(비대칭: 배너 1 + 위 요소 3). 하단에 '어떻게 골랐나' 한 줄.
- **카피:**
  - (배너) **토대 — 데이터 모델·합성키:** 모든 할 일이 사람·사업·활동·작업 키 하나로 역추적
  - STT **4.66%** (기존 대비 약 ¼ · 야외·고령 벤치)
  - Vision **91.3%** *작은 글씨:* 위험통과 0% · 과도반려까지 같이 평가 · 23장 평가셋
  - TTS **1위** (3인 블라인드)
  - (하단 한 줄, 강조) 앱 안에 3모델 비교 모드 + **CSV 로그**로 직접 선정 · 벤더 분산(구글·업스테이지·오픈AI·리턴제로) · 상세 부록 A4·A5
- **비주얼:** 큰 숫자 3개가 시선(4.66%·91.3%·1위 크게). 배너는 받침. 하단 '비교 모드' 줄에 *아주 작은* 3-up 비교 목업이나 CSV 아이콘으로 "직접 만든 도구" 느낌.
- **화면 vs 말로:** *화면엔* 숫자 3개 + 배너 + 비교모드 한 줄. *말로* 풀 것 — Vision은 정확도가 아니라 *두 종류 오류*(위험통과 vs 과도반려)로 나눠 봤다는 점, 비교 모드를 직접 만들어 로그까지 남겼다는 점, "비교 도구·키 설계는 베껴서 못 만든다".
- **노트:** "앱내 비교모드·합성키"가 진위의 핵심 증거 — 하단 한 줄로 분명히. 별도 임팩트 슬라이드 없음(여기 숫자가 그 역할). 발표자 1:15 내레이션 → 슬라이드는 숫자·증거 앵커만.

## S12. 팀 굿바이브
- **목적:** 팀 짧게.
- **레이아웃:** 헤드라인 "어르신이 실제로 쓸 수 있나" + 4인 역할(가로) + 하단 접근성 한 줄.
- **카피:** 박찬호(기획·프론트·AI 연동·백엔드) / 강창희(백엔드·DB·RAG) / 박순선(QA·산출물·UX) / 이동현(초기 기획·문제정의 보조) / (하단) WCAG 2.2 참고 — 글씨·버튼·색 대비 점검
- **비주얼:** 담백하게. 헤드라인이 위, 이름은 작게.
- **노트:** 팀으로 끝내지 않음 — 다음이 비전.

## S13. 비전 · 클로징
- **목적:** 감소 목표 + 북엔드로 닫기.
- **레이아웃:** 상단 "우리가 줄이려는 것" 3항목(목표) + 하단 큰 북엔드 문장.
- **카피:** (목표, 작게) 재촬영 · 반려 · 포기 — *시범에서 측정해 목표 확정* / (북엔드, 대) "매일 기록, 쓰시나요? — **이제, 사진 한 장과 목소리 한 마디로.**"
- **비주얼:** 북엔드 문장이 주인공(S1과 대구). 목표 3항목은 위 작은 띠.
- **노트:** **"줄였다" 금지, "줄이려는 것·측정 예정"** (목표로). 마지막 화면은 북엔드 한 줄로 깔끔하게.
