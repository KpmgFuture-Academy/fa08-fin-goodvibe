# 사업 기본 흐름 (사업 → 활동 → todo → 일지/증빙)

> 현재 데모 컨텍스트: 2026 저탄소 농업 프로그램 시범사업(경종, KK26A001), 서호마을작목반(1000000102).

---

## 1. 4계층 도메인

```
사용자/마을         사업/활동                      기록                      도우미
─────────         ─────────                    ─────────                  ─────────
user_master       project       prj_todo_list   journal      evidence    farm_helper_pair
farmer            prj_group     ────────────►   ────────►    ────────►   notification
village           prj_activity                  prj_journal
ville_group       farm_job
group_member
parcel
```

- **PK 규칙**: `journal` / `evidence` 는 `(user_no, job_date, exec_no[, seq_no])` 복합 PK. `parcel` 은 `(amo_regno, parcel_no)`.
- **화면용 ID**: PK 들을 `-` 로 이어붙인 string. 예: `10000002-20260604-1`.

---

## 2. 사업 참여 등록 (이장님 또는 관리자 수행)

1. 이장님이 `web_user` 의 **사업참여** 메뉴 → 마을 농가를 사업에 등록.
   - `POST /engage/projects/{prj_id}/register` — 농가 등록
2. 활동 선택.
   - `POST /engage/projects/{prj_id}/activities/register`
3. 등록된 활동에 대해 **`prj_todo_list`** 가 자동 생성됨.
   - `POST /engage/projects/{prj_id}/todos/create`
4. 농업인 앱에서 `GET /todo/today` 로 즉시 노출.

## 3. 농업인 기록 흐름

### 3-A. 사진 증빙 (라이브 코칭)

```
HomeScreen 의 todo 카드 클릭
   ↓ PhotoInputScreen
   ↓ PhotoLiveCoachOverlay 자동 mount
      · getUserMedia({video: environment}) — HTTPS/localhost only
      · 3초마다 frame → POST /photo-guard/coach (file, evidence_type, job_cd)
      · backend 가 시행지침 9p 기준 prompt 주입 (PIC2/PIC1 × R0008/RD001/...)
      · status=ok 면 셔터 초록 fill + TTS "잘 보여요, 찍어 주세요"
   ↓ 사용자 셔터
   ↓ POST /evidence/upload
      · evidence_service: 워터마크 + GPS + 역지오코딩
      · diary_rdb: journal + prj_journal 동시 INSERT (parcel_no 포함)
      · evidence_rdb: evidence row
   ↓ CompletionModal (홈화면 룩앤필)
```

### 3-B. 손으로 직접 적기 (+ 사진 첨부)

```
HomeScreen 의 "손으로 직접 적기" → ManualInputScreen
   ↓ 작업 종류 chip + 필지 + 메모
   ↓ (선택) 사진 첨부 카드 → PhotoLiveCoachOverlay
   ↓ "기록 저장하기"
      · linkedEvidenceText = 첨부 evidence_id
      · POST /diary  → linked_evidence_ids 자동 연결
```

### 3-C. 말로 남기기 (STT/TTS 대화)

```
"말로 남기기" → VoiceInputScreen
   ↓ MediaRecorder + 30초 자동 정지 + /ai/stt
   ↓ 인식된 텍스트를 input 에 채움 (자동 전송 X)
   ↓ 사용자 ➤ 누르면 /ai/voice/session/reply → 영농일지 초안
   ↓ 확인 후 /ai/voice/session/finalize → DB 저장
```

## 4. 이장님 검토 흐름

```
web_user/dashboard
   ↓ "오늘 먼저 챙길 일" — laggards top 3 (정렬: 미확인 + 임박)
   ↓ [문자로 알려주기] / [농가 보기]
   ↓
web_user/evidence
   ↓ 검토 필요 사진들 → PATCH /evidence/{id}
   ↓ status: confirmed | retake_required
   ↓ retake_required 시 backend 가 농가에게 notification (content_cd=RETAKE)
   ↓
농업인 앱 → 알림 panel → "다시 찍어주세요" 카드 → PhotoInputScreen
```

## 5. 농사 도와주기 (helper mode)

```
이장님이 web_user/farm-helpers 에서 도우미 ↔ 받을 농가 pair 생성
   POST /admin/farm-helpers
   ↓
양쪽이 농업인 앱에서 동의 (HelperConsentModal)
   ↓ helper_approved_at / recipient_approved_at upsert
   ↓
helper 가 홈에서 "농사 도와주러 가기" → handleToggleHelperMode(true)
   · helperModeOn=true 즉시 → effectiveFarmerIdForApp = recipient
   · HelperModeTransitionScreen ("enter") 2.5초 트랙터 애니메이션
   · 그 사이 recipient 의 todo/사진/일지 fetch 완료
   ↓
helper 가 recipient 의 todo 로 기록 (사진/일지)
   · backend 는 recipient 의 user_no 로 저장
   ↓
"도움 마치기" → handleToggleHelperMode(false)
   · helperModeOn=false 즉시 → 내 농가 데이터 fetch 시작
   · HelperModeTransitionScreen ("leave") 트랙터 반대 방향 2.5초
   · 트랜지션 끝나면 fully loaded 내 홈
```

## 6. 알림 흐름

`notification` 테이블의 `content_cd` 가 분기 키:

| content_cd | 의미 | 농업인 앱 액션 |
|---|---|---|
| `RETAKE` | 이장님이 재촬영 요청 | "사진 찍으러 갈까요?" prompt → PhotoInputScreen |
| `EVID_DEL` | 증빙 삭제됨 | PhotoInputScreen |
| `MANUAL` / `DIA_DEL` | 일지 작성 필요 | "음성으로 기록할까요?" prompt → VoiceInputScreen |
| `TODO_DUE` | todo 임박 | VoiceInputScreen |
| `HLP_INV` | 도우미 초대 | HelperConsentModal 즉시 노출 |
| `HLP_REV` | 도우미 종료 | 별도 액션 없음 (닫기) |

폴링: `LocavilleApp` 이 60초마다 `fetchFarmerUnreadCount` 호출.

## 8. 확장 포인트

- 새 활동 추가 → `farm_job` row 추가 + `todo_rdb._required_evidence_types_by_job` mapping 추가 + `photo_guard_service.JOB_EVIDENCE_TO_CRITERIA` 추가.
- 새 사업 추가 → `project` + `prj_activity` + `prj_todo_list` 시드.
- 새 마을 추가 → `village` + `ville_group` + `group_member` 시드.
